## Tipo de mudança

<!-- Marque o tipo conforme Conventional Commits (alinhado ao prefixo da branch). -->

- [ ] `feat` — nova funcionalidade
- [ ] `fix` — correção de bug
- [ ] `docs` — documentação
- [ ] `style` — formatação, sem mudança de lógica
- [ ] `refactor` — reestruturação interna, sem mudança de comportamento
- [ ] `perf` — melhoria de performance
- [ ] `test` — testes
- [ ] `build` — build/dependências
- [ ] `ci` — integração contínua
- [ ] `chore` — tarefas de manutenção
- [ ] `revert` — reverte um commit anterior

> Correção urgente em produção não é um tipo próprio: use `fix` + a label `priority/critical`.

## Checklist

- [ ] Os testes passam localmente.
- [ ] Adicionei/atualizei testes para a mudança (quando aplicável).
- [ ] Atualizei o README se a mudança impacta setup, requisitos, ambientes ou deploy.
- [ ] Defini 1 ou 2 reviewers conforme a criticidade (2 se `priority/critical` ou se toca segurança/segredos/deploy).
